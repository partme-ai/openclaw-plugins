# OpenClaw 企业级多通道联动架构

## 概述

openclaw-plugins 的企业级目标是：**以 OpenClaw 智能体为核心，打通 IM 渠道、消息队列、知识库、长期记忆，形成多平台信息流闭环**。

本架构文档描述实现这一目标的完整技术方案。

```
    ┌───────────────────────┼───────────────────────┐

    ▼                       ▼                       ▼
┌──────────┐         ┌──────────┐           ┌──────────┐
│ IM 渠道  │         │ 消息队列  │           │ 能力增强 │
│ wecom    │         │ mqtt     │           │ knowledge│
│ wechat   │         │ rabbitmq │           │          │
│ dingtalk │         │ redis    │           │          │
│ qqbot    │         │ rocketmq │           │          │
│ lark     │         │ stomp    │           │          │

└──────────┘         └──────────┘           └──────────┘
```

## 缺失能力

| 能力 | 问题 | 影响 |
|------|------|------|
| **跨渠道消息路由** | 企微消息无法自动转发到 MQ，MQ 消息无法回复到 IM | 业务系统无法追踪对话，无法实现 B2B 联动 |
| **知识库开箱即用** | knowledge 插件存在，但需要 Agent 主动调用工具，用户不会用 | AI 对话缺少企业知识支撑 |
| **长期记忆** | 每次对话从零开始，Agent 不记得用户 | 重复问题反复问，客户体验差 |
| **消息审计** | 消息来了走了，没有统一记录 | 合规场景无法满足 |

## 三流模型

```
flow1: IM 入站流
  用户(企微) → wecom插件 → Agent → agent_end事件
                                          │
                                          ├─→ 回复给企微 (正常)
                                          └─→ router转发到MQ (副本)

flow2: MQ 入站流
  业务系统 → mqtt话题 → mqtt插件 → Agent → agent_end事件
                                          │
                                          └─→ router转发到IM (通知运维)

flow3: 增强流 (每次对话自动触发)
  消息到达 → before_prompt_build事件
              │
              ├─→ 知识库自动搜索 → 注入上下文
              └─→ 长期记忆自动召回 → 注入上下文
```
