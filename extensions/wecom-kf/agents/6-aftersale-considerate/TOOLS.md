# TOOLS.md - Local Notes

Skills define _how_ tools work. This file is for _your_ specifics — the stuff that's unique to your setup.

## What Goes Here

Things like:

- Aftersale script library path or knowledge-base ID (order status, returns, warranty, usage, complaint)
- Company business content path (policy, warranty, returns, FAQ, brand tone)
- Escalation and handoff rules (when to hand to human) — can go here or in MEMORY.md

## Examples

```markdown
### Aftersale Scripts

- Order / delivery: path or KB id
- Returns / warranty: path or KB id
- Complaint / escalation: path or KB id

### Business

- Policy/Warranty/Returns/FAQ: path or link
- Brand tone: (short note)

### Escalation

- Hand to human: (rules)
```

## Why Separate?

Skills are shared. Your setup is yours. Update this when scripts and knowledge change so aftersale replies stay evidence-based.
